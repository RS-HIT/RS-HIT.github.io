# 项目结构说明

## 📁 目录结构

```
RS-HIT.github.io/
├── index.html              # 主页面 - 单页应用入口
├── style.css              # 全局样式文件
├── assets/                # JavaScript模块目录
│   ├── app.js             # 主应用逻辑
│   ├── journal.js         # 学习日志管理模块
│   └── project-manager.js # 项目管理模块
├── templates/             # 日志模板目录
│   ├── journal-development.html   # 开发进展模板
│   ├── journal-learning.html      # 学习心得模板
│   ├── journal-bug-fix.html       # 问题解决模板
│   ├── journal-idea.html          # 想法记录模板
│   └── journal-milestone.html     # 里程碑模板
├── projects/              # 项目详情页目录
│   └── project-detail-template.html  # 项目详情页模板
├── pic/                   # 图片资源目录
└── README.md             # 项目说明文档
```

## 🚀 功能模块

### 1. 主应用 (index.html + assets/app.js)
- **职责**: 单页应用路由、页面切换、全局事件管理
- **特点**: 高内聚的应用入口，统一管理所有页面逻辑

### 2. 项目管理模块 (assets/project-manager.js)
- **职责**: 项目数据管理、项目详情页面渲染
- **特点**: 独立的项目数据处理，支持动态项目内容

### 3. 学习日志模块 (assets/journal.js)
- **职责**: 日志增删改查、统计数据、本地存储
- **特点**: 完整的CRUD操作，数据持久化

### 4. 日志模板系统 (templates/)
- **职责**: 提供标准化的日志格式模板
- **特点**: 可复用的HTML模板，便于用户参考和编辑

### 5. 项目详情页模板 (projects/)
- **职责**: 项目详情页面的独立模板
- **特点**: 可单独使用的项目展示页面

## 🎯 设计原则

### 高内聚
- **模块功能集中**: 每个JavaScript模块专注于单一职责
- **数据封装**: 使用类封装相关数据和方法
- **模板独立**: 每个模板文件包含完整的功能逻辑

### 低耦合  
- **模块独立**: 各模块可独立运行，互不依赖
- **接口清晰**: 通过明确的函数接口进行模块间通信
- **配置分离**: 项目数据和模板内容分离存储

## 📝 使用说明

### 编辑日志模板
1. 进入 `templates/` 目录
2. 选择对应类型的模板文件
3. 直接编辑HTML内容，保存即可

### 添加新项目
1. 在 `assets/project-manager.js` 中的 `defaultProjects` 添加项目数据
2. 或通过页面界面动态添加

### 自定义样式
- 编辑 `style.css` 文件
- 所有样式统一管理，便于维护

## 🔧 技术栈

- **前端**: HTML5 + CSS3 + 原生JavaScript
- **架构**: 单页应用 (SPA)
- **存储**: LocalStorage
- **模块化**: ES6 Classes + Module Pattern
- **响应式**: CSS Grid + Flexbox
